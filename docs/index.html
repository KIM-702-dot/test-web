Should you use Types or Interface?

Sa TypeScript, both type at interface ay ginagamit para mag-define ng shapes ng data. Pero, paano mo malalaman kung alin ang gagamitin?

Kung simple object lang ang gusto mong i-define, interface ang mas readable and extendable. Pero kung gusto mong gumawa ng union types o complex types like tuples, dun papasok ang type.

Example gamit ang interface:

interface User {
  name: string;
  age: number;
}

Example gamit ang type:

type User = {
  name: string;
  age: number;
};

Technically, halos pareho lang sila sa karamihan ng cases. Pero sa mga mas malaking codebase, mas preferred ang interface dahil mas madali itong i-extend.

At tandaan, may mga limitations ang interface tulad ng hindi ito puwedeng gamitin sa union types. Kaya kung kailangan mong gumawa ng type na pwedeng string o number, dapat type ang gamitin.
